# dak+gord-system – Run Types v1

## 1. `datei_lesen`
Zweck:
- Datei lesen
- Fokus setzen
- Dossier aktualisieren

Typische Eingabe:
- `lesen vision4`

Typische Artefakte:
- Fokuskontext
- Verdichtung
- Agentdatei

## 2. `anschlussantwort`
Zweck:
- Folgefrage zu aktiver Fokusdatei beantworten

Typische Eingaben:
- `was weisst du darüber`
- `geh noch tiefer`

Typische Artefakte:
- Anschlussantwort
- Agentdatei-Verlauf
- Fokusfortschritt

## 3. `neugier_scan`
Zweck:
- Werkraum-Neugier auslösen
- Spur und Wochenlog schreiben

Typische Artefakte:
- Werkraum-Spur
- Wochenlog
- Ticker-Log

## 4. `vision_cycle`
Zweck:
- Vision-Dateien in Ausschnitten durchlaufen
- Vision-Spuren schreiben

Typische Artefakte:
- Vision-Spur
- Wochenlog
- Cursor-Fortschritt

## 5. `verdichtung_refresh`
Zweck:
- Verdichtung oder Dossier unabhängig aktualisieren

## 6. `maintenance`
Zweck:
- spätere Konsolidierung, Bereinigung, Rebuilds
